Hospital Readmission Analysis (SQL)
Project Summary:
I analyzed 101,766 diabetic patient records from 130 US hospitals to find out why patients get readmitted. This project covers the full process: cleaning messy clinical data, analyzing risk factors, and preparing the data for professional dashboards.

🛠️ What I Did
Data Cleaning: Cleaned up "dirty" data by replacing ? markers, removing duplicates, and dropping columns that were mostly empty.

Feature Engineering: Converted text-based age groups into numbers and created medication categories so the data is actually usable for charts.

Advanced SQL: Used CTEs, Window Functions, and Temp Tables to calculate readmission rates by age, race, and medical specialty.

BI Ready: Wrapped everything into a SQL View so it can plug directly into Power BI or Tableau.

🔍 Key Insights
Diagnosis Count: Patients with 9+ diagnoses are much more likely to return within 30 days.

Medication Changes: Patients whose insulin dosage was adjusted during their stay had a higher risk of coming back.

Stay Length: Identified how the number of days spent in the hospital directly correlates with the chance of readmission.

💻 Skills
SQL (T-SQL): CTEs, Window Functions, Data Cleaning, ETL, Views.

Analytics: Healthcare data trends and patient risk-factor analysis.
